- **Core Abstractions (`core.py`)**: Defines an abstract base class `BaseFunction` with runtime context injection (`ctx`) and logging. Specialized stream operators (`MapFunction`, `FilterFunction`, `FlatMapFunction`, etc.) extend this base, enforcing a consistent execution contract via the `execute` method. A `wrap_lambda` utility allows dynamic conversion of standard Python callables into these typed function classes.
- **Configuration Subsystem (`config/`)**: Implements `SagePorts` for deterministic service port assignment (with WSL detection) and `SageUserPaths` for XDG-standard directory management (config, data, state, cache), including legacy migration support.
- **Model Registry (`model_registry/`)**: Manages local LLM model assets using a JSON manifest (`metadata.json`) for metadata tracking. It integrates with `huggingface_hub` for downloads and provides atomic manifest updates via temporary file replacement.
- **Observability (`logging.py`, `debug.py`)**: Offers a `CustomLogger` wrapper around Python's `logging` module with global debug toggles, and a `PrintSink` for stream debugging.
- **Dependency Direction**: Higher-level modules (`model_registry`) depend on `config` for path resolution, while `core` remains independent of infrastructure concerns.
