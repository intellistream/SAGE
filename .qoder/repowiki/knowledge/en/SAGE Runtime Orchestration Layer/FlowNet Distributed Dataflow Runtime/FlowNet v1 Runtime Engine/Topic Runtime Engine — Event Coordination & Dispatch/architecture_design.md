- Entry point is `TopicAPI` (`topic_api.py`), which composes internal managers: `FlowProcessCatalog`, `TopicRoutingDirectory`, `FlowProgramRoutingDirectory`, `TopicCoordinatorRegistry`, `TopicSubscriberRegistry`, `EventGroupLedgerManager`, `SubscriberProgressManager`, `TopicEventDispatch`, and `TopicControlDispatch`.
- Routing layer (`routing_directory.py`) holds immutable `TopicRouteView` snapshots keyed by `(topic_uri, epoch)` and tracks switch-window markers via `observe_admission` / `observe_completion`; `FlowProgramRoutingDirectory` maps `FlowProgramRef` to owner addresses.
- State registries (`coordinator_registry.py`, `subscriber_registry.py`) are lazy, thread-safe stores keyed by composite tuples `(topic_uri, epoch)` and `(topic_uri, epoch, subscriber_id)`, each with idle GC based on `updated_at` timestamps.
- Dispatch helpers (`event_dispatch.py`, `control_dispatch.py`) implement a uniform local-vs-forward pattern: resolve route from `TopicRoutingDirectory`, compare coordinator address to local address, then either apply locally or build a forward intent dict.
- Communication bridge (`comm_bridge.py`) registers handlers on `V1CommHub` for five wire ops (one data-plane event forward, four control-plane forwards) and exposes `send_topic_forward_intent_via_comm` to emit intents as one-way messages.
- Normalization utilities (`normalization.py`) centralize URI canonicalization (`flownet://cluster/scope/topic/name`), non-empty string validation, and non-negative integer coercion, used across all modules.
