- **Contract-Driven Boundary**: The `contracts` module defines immutable schemas (e.g., `FlowProgramSubmitContract`, `RecoveryPolicy`) that enforce strict validation across the client submission, compiler transformation, and runtime execution layers.
- **Unified Entry Point**: The top-level `__init__.py` aggregates APIs from `api` (DSL), `compiler` (build), `client` (session/cluster), and `runtime` (execution) into a single developer-facing interface, hiding internal modular complexity.
- **CLI Orchestration**: The root `cli.py` acts as a command router, delegating node and cluster lifecycle operations to the `node` module while maintaining a consistent argument forwarding pattern for infrastructure management.
- **Runtime-Client Separation**: A clear architectural split exists between the `client` module (handling session bootstrapping, cluster context resolution, and remote inspection) and the `runtime` module (managing local actor execution, topic coordination, and multi-plane communication).
