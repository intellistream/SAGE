### 改进 (Improved)

#### 安装进度显示 - 更详细的下载信息

**问题**: 用户在运行 `quickstart.sh` 安装时，只能看到下载文件数量，无法了解：

- 文件大小是多少
- 下载速度有多快
- 为什么安装这么慢

**改进内容**:

1. **实时显示文件大小和下载速度**

   - 从 pip 输出中提取文件大小（kB/MB/GB）
   - 统一转换为 MB 并累计
   - 实时计算并显示下载速度（MB/s）
   - 示例：`⬇ 下载中... [43 个文件, 325.6 MB 已下载, 0.52 MB/s]`

1. **每60秒显示网络性能评估**

   - 自动评估网络性能等级：
     - 🟢 快速网络: > 2 MB/s
     - 🔵 正常网络: 0.5-2 MB/s
     - 🟡 慢速网络: < 0.5 MB/s
   - 示例：`[已运行 623s，已下载 325.6 MB @ 0.52 MB/s | 正常网络 (0.5-2 MB/s)]`

1. **慢速网络智能提示**

   - 当平均速度 < 0.3 MB/s 时，自动显示优化建议
   - 示例：`提示: 下载速度较慢（0.28 MB/s），可能需要检查网络连接或使用镜像源`

1. **完成后显示总体统计**

   - 安装完成后汇总信息：文件总数、总大小、总耗时、平均速度
   - 示例：`📊 安装统计: 共 51 个文件, 328.4 MB, 耗时 684s, 平均 0.48 MB/s`

**影响范围**:

- ✅ 自动应用于所有使用 `quickstart.sh` 的安装
- ✅ 自动应用于所有调用 `log_pip_install_with_verbose_progress` 的脚本
- ✅ 向后兼容，不影响现有功能

**技术实现**:

- 文件: `tools/install/ui/logging.sh`
- 函数: `log_pip_install_with_verbose_progress`
- 依赖: `bc` 命令（用于浮点数计算，大多数系统自带）

**测试**:

- 测试脚本: `tools/install/test_progress_display.sh`
- 使用示例: `tools/install/examples_progress_display.sh`
- 详细文档: `tools/install/PROGRESS_DISPLAY_IMPROVEMENT.md`
